   ┌─ users table
   ├─ id INTEGER PRIMARY KEY AUTOINCREMENT
   ├─ username TEXT UNIQUE NOT NULL (case-insensitive)
   ├─ email TEXT UNIQUE NOT NULL (case-insensitive)
   ├─ passwordHash TEXT NOT NULL (bcrypt hash)
   ├─ name TEXT NOT NULL
   ├─ experience TEXT DEFAULT 'Not specified'
   ├─ createdAt DATETIME DEFAULT CURRENT_TIMESTAMP
   ├─ updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP
   └─ Indexes on username and email for fast lookups

2. scripts/migrate-to-db.js (100 lines)
   ────────────────────────────────────────────────────────────────────────
   MIGRATION SCRIPT - Moves existing accounts from JSON to SQLite
   
   Features:
   • Reads accounts.json
   • Hashes all existing plain text passwords with bcrypt
   • Imports accounts into SQLite database
   • Creates backup of original accounts.json
   • Handles duplicate accounts gracefully
   • Provides detailed migration report
   
   Usage: node scripts/migrate-to-db.js
   
   Reports:
   • Number of accounts migrated
   • Number skipped (already in DB)
   • Backup file location

================================================================================
                        VALIDATION RULES
================================================================================

REGISTRATION FORM:

1. FULL NAME (required)
   • Any text, required
   • No specific format restrictions

2. USERNAME (required)
   • Minimum 3 characters
   • Only alphanumeric, underscore (_), hyphen (-)
   • Must be unique (case-insensitive)
   • Real-time availability check provided

3. EMAIL (required)
   • Valid email format (standard regex)
   • Must be unique (case-insensitive)
   • Real-time availability check provided

4. EXPERIENCE (optional)
   • Free text field
   • Defaults to "Not specified" if empty

5. PASSWORD (required)
   • Minimum 8 characters
   • Must contain: uppercase, lowercase, digit, special character
   • NOT PLAIN TEXT - BCRYPT HASHED
   • Real-time strength indicator shows: Weak/Fair/Good
   
   Strength Algorithm:
   • Weak: < 8 chars OR missing requirements
   • Fair: 3 of 4 requirements met
   • Good: All 4 requirements met

6. CONFIRM PASSWORD (required)
   • Must match password field exactly
   • Real-time matching indicator
   • Cannot submit if passwords don't match

LOGIN FORM:

1. USERNAME
   • Case-insensitive matching
   • Returns "Invalid credentials" if not found

2. PASSWORD
   • Compared against bcrypt hash using bcrypt.compare()
   • Returns "Invalid credentials" if mismatch
   • Secure timing-safe comparison

                          SUPPORT & MAINTENANCE
================================================================================

COMMON ISSUES:

Q: Users can't log in after update?
A: Run migration script: node scripts/migrate-to-db.js

Q: Database locked error?
A: SQLite is single-writer. Stop other processes using port 5000.

Q: Migration script shows "No accounts to migrate"?
A: accounts.json is empty or doesn't exist. Database initialized fresh.

Q: How to backup user data?
A: Copy referees.db file to backup location.

Q: How to reset passwords?
A: Currently no reset feature. Users can create new account.
  (Future: Add password reset endpoint)

Q: Performance degradation?
A: Check database indexes exist. Run: SELECT * FROM sqlite_master WHERE type='index';
